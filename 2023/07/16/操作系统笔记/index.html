操作系统

引论

  • 操作系统的目标
    • 方便性
    • 有效性
    • 可扩充性
    • 开放性
  • 推动操作系统发展的主要动力
    • 不断提高计算机资源利用率
    • 方便用户
    • 器件的不断更新迭代
    • 计算机体系结构的不断发展
    • 不断提出新的应用需求
  • 单道批处理系统
    • 内存中只有一个程序
    • 优点：
      • 单道性
      • 封闭性、独占性
      • 顺序性
    • 缺点
      • 系统资源利用不充分
  • 多道批处理系统
    • 中断技术出现
    • 实现：
      • 用户提交的作业形成一个队列（后备队列）由作业调度程序按一定算法选出作业执行
    • 优缺点：
      • 资源利用率高
      • 系统吞吐量大
      • 平均周转时间长
      • 无交互能力
    • 需要解决的问题：
      • 处理机争用问题
      • 内存分配和保护问题
      • I/O设备分配问题
      • 文件的组织和管理问题
      • 作业管理问题
      • 用户与系统的接口问题
  • 分时系统
    • cpu快于I/O，cup可以和I/O同时工作
    • 分时系统允许多个用户同时通过自己的终端，以交互方式使用资源，共享计算机
    • 需要处理的问题：
      • 及时接收
        • 扫描
      • 及时处理
        • 作业直接进入内存
        • 采用轮转运行方式
    • 特征：
      • 多路性
      • 独立性
      • 及时性
      • 交互性
  • 实时系统
    • 能够及时响应外部事件的请求，在规定时间内处理，并控制所有实时任务协调一致运行
    • 类型：
      • 工业控制、信息查询、多媒体、嵌入式
    • 任务类型：
      • 周期性实时任务、非周期性实时任务
      • 硬实时任务、软实时任务
    • 实时系统与分时系统的比较
      • 多路性
      • 及时性
      • 交互性
      • 可靠性
      • 独立性
  • 操作系统的发展
    • 单用户单任务—>单用户多任务—>多用户多任务
  • 操作系统的基本特性
    • 并发
      • 并行：多个事件同一时刻发生
      • 并发：多个事件同一时间间隔内发生
    • 进程
    • 共享
    • 虚拟
      • 时分复用
      • 空分复用
    • 异步
      • 进程的不确定性，不可预知的
    • 处理机管理功能
      • 对进程控制
      • 对通信控制
      • 调度
        • 作业调度 外存—>内存
        • 进程调度 对cup资源调度
    • 储存器管理功能
      • 内存分配
        • 静态分配
        • 动态分配
      • 内存保护（互不干扰）
        • 确保每个用户程序都仅在自己的内存空间运行
        • 绝不允许用户程序访问操作系统的程序和数据，也不允许转移到非共享的用户程序中执行
      • 地址映射
        • 逻辑地址—>物理地址
      • 内存扩充
        • 借助 虚拟储存技术
        • 请求调度功能，用到时再调入
        • 置换功能，腾出内存空间
    • 设备管理功能
      • 缓冲管理，cup和IO设备间缓冲
      • 设备分配
      • 设备处理 又叫 设备驱动程序
    • 文件管理功能
      • 文件储存空间管理
      • 目录管理
      • 文件的 读写管理 和 保护
  • 微内核OS结构
    • 小内核
    • 客户/服务器模式
    • 机制与策略分离 原理
    • 面向对象技术
  • 微内核基本功能
    • 进程（线程）管理
    • 低级储存器管理
    • 终端 和 陷入 处理
    • 作为机制写入内核——机制在执行时不可被打断
  • 微内核优点
    • 提高了系统的可扩展性
    • 增强了系统的可靠性
    • 可移植性强
    • 提供了对分布式系统的支持
    • 融合了面向对象技术
  • 微内核缺点
    • 效率降低：：服务请求时多次上下文切换，多次交互，用户态和内核态切换

进程的描述与控制

  • 前趋图 和 进程执行

    • 有顺序性

  • 程序的顺序执行

    • 系统资源利用率低
    • 特征：
      • 顺序性
      • 封闭性 —–> 独占全机资源
      • 可再现性 —–> 初始条件相同，可重复执行处相同结果

  • 程序并发执行

    • 提高系统资源利用
    • 只有不存在前趋关系的程序才能并发执行
    • 特征：
      • 间断性 —–> 相互合作，相互制约，共享系统资源
      • 失去封闭性
      • 不可再现

  • 进程的描述

    • 进程的定义

    • 一个程序可以分为多个进程

    • 程序段 + 相关数据段 + pcb == 进程实体（进程映像）

    • 创建进程 是 创建进程实体的pcb

    • 撤销进程 是 撤销进程的 pcb

    • 进程是程序的一次执行

    • 进程是一个程序及其数据在cup上顺序执行时所发生的活动

    • 进程是具有独立功能的程序在一个数据集合上运行的过程，是系统进行资源分配和调度的独立单位

    • 进程特征（进程有一定的生命周期，程序是静态的）

      • 动态
      • 并发
      • 独立
      • 异步

    • 进程三种基本状态
